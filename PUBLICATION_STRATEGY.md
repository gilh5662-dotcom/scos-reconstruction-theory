# Publication Strategy

Date prepared: 2026-06-29

Author / inventor: Gilberto Herrera-Olaguez

This document explains how this repository should be treated when publishing
SCOS material. It is not legal advice.

## Core Position

This repository is a technical disclosure record. It is not a patent
application, provisional patent application, peer-reviewed paper, journal
publication, or grant of rights.

The current publication stance is:

1. publish enough technical detail to create an author-controlled disclosure
   record;
2. reserve copyright and patent rights;
3. avoid open-source licensing before a patent strategy is chosen;
4. use the repository as supporting material for counsel, reviewers, and future
   filings;
5. file a provisional or nonprovisional patent application promptly if patent
   protection is desired.

## What GitHub Publication Does

A public GitHub repository can provide a timestamped, publicly accessible
technical disclosure. That may help establish authorship and can become prior
art against later attempts by others to claim the same disclosed material.

To be more useful as a disclosure record, this repository includes:

1. a descriptive README;
2. a rights-reserved license;
3. a dated public disclosure notice;
4. a patent disclosure draft;
5. an architecture diagram;
6. a runnable simulation.

## What GitHub Publication Does Not Do

Publishing this repository does not:

1. file a patent application;
2. create "patent pending" status;
3. guarantee a 12-month grace period in every country;
4. guarantee patentability;
5. prove reduction to practice for every claimed embodiment;
6. grant permission for others to commercialize the work;
7. replace a patent attorney.

## Recommended Breakthrough-Tech Path

For a potentially patentable breakthrough, the safer order is usually:

1. write an invention disclosure;
2. identify inventors and ownership;
3. run a prior-art search;
4. file a provisional patent application before broad public release;
5. publish supporting technical material after filing, if publication is still
   desired;
6. refine claims and embodiments during the provisional year;
7. file a nonprovisional and any international applications before deadlines.

This repository currently supports steps 1 and 5. It does not complete step 4.

## Patent Timing Risk

In the United States, inventor-originated public disclosures may be eligible for
a limited grace period. Other jurisdictions may be stricter or may not preserve
rights after public disclosure. Relying on a grace period is riskier than filing
before publication.

If patent protection is important, contact a patent attorney quickly and show
them:

1. `README.md`;
2. `PATENT_DISCLOSURE.md`;
3. `ROADMAP.md`;
4. `simulations/scos_physical_medium.py`;
5. the Git commit history and public repository URL.

## Current Repository License Stance

This repository intentionally uses an all-rights-reserved license instead of
MIT, Apache, GPL, or Creative Commons.

Reason: open-source and Creative Commons licenses can grant reuse rights before
the patent strategy is settled. The current license is meant to preserve rights,
not encourage unrestricted reuse.

## Publication Checklist

Before making the repository public:

1. confirm the legal name is correct;
2. confirm all inventors are listed if there are co-inventors;
3. confirm the rights notice is correct;
4. confirm no confidential third-party material is included;
5. confirm no employer, contractor, school, or organization owns any part of the
   invention;
6. decide whether to file a provisional patent application first;
7. save a local archive of the repository and commit hash after publication.
